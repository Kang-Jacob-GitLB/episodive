Loading & Error

Loading indicators, full-screen loading states, and error displays

core/designsystem/src/main/kotlin/.../component/LoadingWheel.kt, LoadingScreen.kt, ErrorScreen.kt

LoadingWheel

A spinning loading indicator using Material3 ContainedLoadingIndicator. Provides visual feedback during asynchronous operations. Compact size suitable for inline or overlay usage.

Showcase

Spinning indicator
On dark surface

Compose Example

LoadingWheel(
    modifier = Modifier.size(48.dp)
)

LoadingScreen

A full-screen centered wrapper around LoadingWheel. Fills all available space with the loading indicator centered both horizontally and vertically. Used as the primary loading state for feature screens.

Showcase

Screen Title
Full-screen loading state
Loading within a container

Compose Example

// Typically used in MVI state handling:
when (state) {
    is MyFeatureState.Loading -> LoadingScreen()
    is MyFeatureState.Success -> SuccessContent(state.data)
    is MyFeatureState.Error -> ErrorScreen(state.message)
}

ErrorScreen

A centered error message display. Shows the error text in error color, centered within the available space. Used as the error state in the MVI pattern across all feature screens.

Showcase

Podcasts
Failed to load podcasts. Please check your connection.

Short error message

Compose Example

ErrorScreen(
    message = "Failed to load podcasts. Please check your connection."

// In MVI pattern:
sealed interface HomeState {
    data object Loading : HomeState
    data class Success(val podcasts: List<Podcast>) : HomeState
    data class Error(val message: String) : HomeState
}
