Screen Overview

The Clip screen presents podcast soundbites in a TikTok-style vertical swipe interface using VerticalPager. Each page displays an EpisodeClipItem with a blurred background, album art, episode information, and playback controls. It uses the @Player(EpisodivePlayers.Clip) ExoPlayer instance, separate from the main player. The screen is lifecycle-aware: it pauses on background and resumes on foreground. Clips auto-play on entry and auto-advance when complete.

Dual Player System

The Clip screen uses @Player(EpisodivePlayers.Clip) qualified PlayerRepository, a separate ExoPlayer instance from the main player. This allows clips to play independently without interfering with the main episode playback. The ClipPlayerState is composed from playerRepository.playback, .progress, and .isPlaying flows.

Navigation

Bottom Nav: Clip tab → ClipScreen → PodcastRoute (tap podcast name)

The Clip screen is one of the four main bottom navigation tabs. It is accessible directly from the navigation bar. Tapping a podcast name within a clip navigates to the Podcast Detail screen via ClipEffect.NavigateToPodcast. Tapping the clip itself triggers ClickEpisode which plays the episode in the main player.
