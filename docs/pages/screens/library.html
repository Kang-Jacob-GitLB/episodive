Library

User's personal collection with section filters and full-text search

Screen Structure

The library screen provides a filterable view into the user's data: played episodes, liked episodes, followed podcasts, and preferred categories. A FindBar at the top enables full-text search across the library. A horizontal SectionFilter chip row switches between 5 views.

Paging UI Models

Paginated sections use sealed interfaces with date separators inserted between items:

PlayedUiModel

  • SeparatorModel(date: String)
  • EpisodeModel(episode: Episode)

LikedUiModel

  • SeparatorModel(date: String)
  • EpisodeModel(episode: Episode)

FollowedUiModel

  • SeparatorModel(date: String)
  • PodcastModel(podcast: Podcast)

Navigation Flow

Library
→
ClickPodcast
→
Podcast Detail

The Library screen is one of the 4 bottom navigation destinations. The only navigation effect is NavigateToPodcast, which routes to the Podcast Detail screen. Episode actions (play, resume, like/unlike) are handled in-place via use cases.

When the find bar has a non-empty query, library data is filtered through FindInLibraryUseCase using full-text search, replacing the default data with matched results.
